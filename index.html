1. CSS (Chirp Spread Spectrum)

Cómo LoRa logra inmunidad al ruido y un gran alcance?

La tecnología LoRa utiliza una técnica de modulación patentada llamada Chirp Spread Spectrum (CSS). Funciona enviando pulsos (chirps) cuya frecuencia aumenta o disminuye progresivamente a lo largo del tiempo.

  • Inmunidad al ruido: Al utilizar estos patrones de barrido de frecuencia, el receptor puede distinguir la señal de LoRa incluso cuando esta se encuentra por debajo del nivel de ruido de fondo (SNR negativo). Es como reconocer una melodía de silbido específica en medio de una habitación ruidosa.
  • Gran alcance: Se logra gracias a un ancho de banda relativamente bajo y un presupuesto de enlace (Link Budget) muy alto. Esto permite que la señal viaje decenas de kilómetros en áreas rurales sin disiparse por completo.

2.Arquitectura LoRaWAN

Cuál es la diferencia entre LoRa y LoRaWAN?

  • LoRa: Es la capa física (Radiofrecuencia). Es la forma en que las ondas electromagnéticas se transmiten por el aire.
  • LoRaWAN: Es la capa de control de acceso al medio (MAC). Es el protocolo de red y la arquitectura del sistema que dicta cómo se comunican los dispositivos, cómo se asegura la red y cómo se enrutan los datos.

Arquitectura de la Red

El ecosistema funciona con una topología de "estrella de estrellas":

  1. Nodos Finales (Sensores): Dispositivos que recopilan datos y los envían vía LoRa.
  2. Gateways (Pasarelas): Actúan como puentes transparentes. Reciben las ondas de radio de los nodos y las reenvían a Internet mediante conexiones IP estándar (Wi-Fi, Ethernet, 4G).
  3. Network Server: El cerebro de la red. Elimina mensajes duplicados (si varios gateways escucharon el mismo sensor), gestiona la seguridad y controla la velocidad de los datos.
  4. Application Server: El destino final donde los datos se descifran, se procesan y se muestran al usuario final.

3. Clases de Dispositivos: Energía vs Latencia

Cuándo priorizar ahorro de energia frente a latencia de bajada?

LoRaWAN define tres clases de dispositivos para adaptarse a diferentes necesidades, equilibrando siempre el consumo de batería y la rapidez con la que el dispositivo puede recibir órdenes de la red (latencia de bajada).

Clase A (Battery Powered)

Prioridad: Máximo ahorro de energía.

El dispositivo duerme casi todo el tiempo. Solo abre dos breves "ventanas de escucha" inmediatamente después de enviar un mensaje. Si la red quiere decirle algo, debe esperar a que el sensor hable primero.

Clase B (Beacon)

Prioridad: Equilibrio.

Además de las ventanas de la Clase A, abre ventanas de escucha programadas a intervalos regulares. Requiere sincronizarse con el Gateway usando balizas (beacons). Gasta más batería que la Clase A.

Clase C (Continuous)

Prioridad: Mínima latencia de bajada.

El dispositivo está escuchando continuamente. Si la red le envía un mensaje, lo recibe al instante. Debido al alto consumo, estos dispositivos suelen estar conectados a la red eléctrica continua.
